# 1. Initial Project Scaffolding

This document outlines the first step in building the Agentic RAG framework: laying out the physical folder structure and creating the foundational configuration files across the Client, Server, and Infrastructure layers.

## Objectives
- Create the core directory structure as defined in `docs/architecture/03-folder-structure.md`.
- Initialize the Python virtual environment for the central server.
- Initialize the React (Vite) application for the UI.
- Set up the base `docker-compose.yml` for the local infrastructure (Qdrant, MinIO, Langfuse).

---

## Execution Steps

### Step 1: Create the Directory Tree
Create the empty foundational directories to match the architecture.
- Create `src/client/csharp-bridge`
- Create `src/server/api`, `src/server/agents`, `src/server/tools`, `src/server/rag`
- Create `src/ingestion/parsers`, `src/ingestion/chunking`, `src/ingestion/storage`
- Create `docker/minio`, `docker/qdrant`, `docker/langfuse`

### Step 2: Initialize Python Server Environment
Set up the `src/server` and `src/ingestion` dependencies.
- Navigate to `src/server`
- Create a virtual environment (`python -m venv venv`)
- Create a `requirements.txt` with initial dependencies:
  - `fastapi`
  - `uvicorn`
  - `pydantic`
  - `google-generativeai`
  - `qdrant-client`
  - `docling` (temporarily added here or in ingestion root)
  - `langfuse`
  - `python-dotenv`

### Step 3: Initialize React UI Environment
Initialize the Vite React project inside `src/client/react-ui`.
- Navigate to `src/client`
- Run `npm create vite@latest react-ui -- --template react-ts`
- Clean up default Vite boilerplate (remove default CSS/App content).
- Install TailwindCSS and shadcn/ui dependencies as explicitly requested in `CLAUDE.md`.

### Step 4: Infrastructure Automation (Docker Compose)
Create the `docker/docker-compose.yml` to spin up the local observability and vector DB stack.
- Define a `qdrant` service.
- Define a `minio` service (with standard default admin credentials for local dev).
- Define the `langfuse` services (Langfuse server + Postgres DB dependency).

---

## Validation / Testing

To verify this plan was executed correctly, the agent must perform the following checks:

1. **Tree Validation:** Run `tree src` and verify the directory structure exactly matches the plan constraints.
2. **Python Env Validation:**
   - Activate `src/server/venv`.
   - Run `pip install -r src/server/requirements.txt` to ensure there are no dependency resolution conflicts.
3. **React App Validation:**
   - Navigate to `src/client/react-ui`
   - Run `npm install` followed by `npm run build` to ensure the clean Vite template compiles without TypeScript or Vite configuration errors.
4. **Docker Validation:**
   - Navigate to `docker/`
   - Run `docker compose config` to validate the YAML syntax of the newly created `docker-compose.yml` file is correct before attempting to spin up heavy containers.
